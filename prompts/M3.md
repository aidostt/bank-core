# Milestone 3: Proof + delivery

Prereq: M2 accepted.

Build:
1. load/k6: `transfers.js` (login, POST /v1/transfers with unique idempotency keys,
   poll GetTransfer) and `reads.js` (GET /v1/accounts). Stages to find laptop
   capacity; thresholds p99<500ms transfer, p95<100ms reads. Run against compose,
   commit summary output + write results table into README.
2. `make chaos`: toxiproxy scenario script — start transfer burst, cut
   transfer↔ledger proxy for 10s mid-burst, restore, wait for recovery worker,
   assert all transfers terminal (COMPLETED/FAILED, none stuck), verify-ledger
   clean. Print a narrative log; paste into README.
3. deploy/helm: umbrella chart `bank-core` with a library subchart (deployment/
   service/configmap/secret templates, probes from /healthz /readyz, resources,
   HPA stub) + per-service values; infra (postgres, redpanda, redis) as subcharts
   or pinned dependencies; `make helm-deploy` → k3d cluster create + install +
   smoke curl. Document in docs/deploy.md.
4. .github/workflows/ci.yml: jobs lint (golangci-lint, buf lint, buf breaking
   against main), test (unit+integration with services containers), build (docker
   build all), e2e (compose up + tests/e2e) — path filters for speed.
5. Root README: what/why, architecture diagram (mermaid), 3-command quickstart,
   demo transcript, k6 results table, chaos transcript, screenshots (Grafana,
   Jaeger), links: ADR index, service docs, PLAN, roadmap. Tone: engineering
   report, no marketing.
6. Final pass: every ADR still true; every service doc matches code; roadmap
   updated; remove dead code; `git log` tells a clean story.
